The input string gets converted to a Librdf\+Node automatically. An arbitrary number of locations are allowed. The location is added to the back of a vector containing the Resources. Left most elements of this vector represent larger physiological locations which get smaller as the index of this vector increases via use of the \char`\"{}is\+Part\+Of\char`\"{} predicate. For example, a cytosolic TGFb molecule in a dermal fibroblast could have a locations vector first containing a reference to skin, then dermis, then fibroblast, then cytosol. \mbox{\hyperlink{classThis}{This}} ends up being cytosol$<$is\+Part\+Of$>$fibroblast$<$is\+Part\+Of$>$dermis$<$is\+Part\+Of$>$skin.

See Librdf\+Node documentation for more details of valid input strings.
