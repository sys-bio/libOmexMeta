equality operator. 

iterates over this stream and looks for containment in rhs stream, and then does the same in reverse.

Maximum complexity\+: O(n$^\wedge$2). Minimum complexity O(1), when lhs and rhs are different sizes. Note on complexity -\/ this algorithm is quite expensive. First, if not same size, then return false. So the rest is assuming the size of lhs and rhs are equal. iterate over lhs. For each of the n elements element we search through n rhs -\/-\/$>$ n$^\wedge$2 Then we iterate over rhs. For each of the n elements we search through n lhs elements -\/-\/$>$ n$^\wedge$2 So we have O(2n$^\wedge$2), but since we ignore multiplicitive and additive constants, its just O(n$^\wedge$2)
