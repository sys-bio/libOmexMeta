◆ resultsAsMap()

ResultsMap omexmeta::Query::resultsAsMap ( )

returns query results in a map where keys are the variable names used in query and values are vectors of values for results.

This is the only supported mechanism for ascertaining query results programatically, without needing to read/write to/from file. This method uses an iterator from librdf. When it gets to the end of the iterator, the iterator is "used up". Therefore at the end of collecting the results as a map, the original query is re-run, so that the presently instantiated Query object is not rendered useless.

If the query looks like this:

std::string query_string = "SELECT ?x ?y ?z
"WHERE {"
" ?x ?y ?z"
"}";
Query query(query_string);

The column names will be x, y and z and the results will be string representations of all subjects (?x), predicates (?y) and resources (?z) in a vector of size n, where n is the number of query results.
