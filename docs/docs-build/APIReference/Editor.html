class omexmeta::Editor¶

Add or change annotations in xml.

Public Functions

Editor(std::string xml_or_file, bool create_ids, const LibrdfModel &model, NamespaceMap &ns_map, UriHandler &uriHandler, bool generate_new_metaids = false, bool sbml_semantic_extraction = true)¶

constructor for Editor.

The

Editor is usually instantiated from the RDF::to_editor class which automatically takes care of the
Parameters

  • xml_or_file: The valid xml content for annotation OR the path to the file on disk containing this content

  • generate_new_metaids.: Autogenerate metaids for xml element that do not already have them

  • sbml_semantic_extraction.: When

  • xml_or_file: is determined to be sbml, automatically extract some information regarding species and reactions. This option is ignored when

  • xml_or_file: is not sbml. Default is true.

  • model: a reference to the current model (owned by RDF).

  • nm_map: a set of namespaces for current xml

Parameters

  • model: and

  • ns_map: arguments. When instantiated, the editor class automatically reads the XML and adds metaids to some or all of the elements, depending on the XML type parameter. If the user specifies that they are annotating an SBML model, the elements returned by OmexMetaSBML::getValidElements() are given metaids (if not exist), while if the user chooses cellml the elements returned by OmexMetaCellML::getValidElements() are used. If the type is unknown, then all elements are given metaids.

~Editor() = default¶

We no longer required to free the triples_ used by Editor since they are created and freed inplace - i.e. locally, not at the class scope.
